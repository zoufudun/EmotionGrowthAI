### 💡 修改代码后，如何更新这两个分支？

为了方便您后续维护，每次修改代码后，您可以按照以下 **四个标准步骤** 在命令行中运行，来一键同步和部署网页：

#### 步骤 1：本地打包构建
每次修改完前端代码（如修改了学校列表、修改了组件逻辑等），必须先重新编译生成最新的网页资源：
```bash
npm run build
```

#### 步骤 2：提交本地代码修改
将本地的修改存入 Git 本地仓库中：
```bash
git add .
git commit -m "您的更新说明（例如：更新了学校列表）"
```

#### 步骤 3：推送源代码到远程分支
将写好的源代码推送到远程的开发分支（`master`）和默认分支（`main`）：
```bash
git push origin master
git push origin master:main
```

#### 步骤 4：将编译后的网页部署到 GitHub Pages
使用 Git 子树命令，把编译好的 `dist` 文件夹内容推送到 `gh-pages` 分支进行部署：
```bash
git subtree push --prefix dist origin gh-pages
```

运行完步骤 4 后，等待 1-2 分钟，GitHub Pages 就会自动拉取最新打包的文件并更新上线了。如果有延迟，可以使用**无痕模式**访问网页查看最新效果。

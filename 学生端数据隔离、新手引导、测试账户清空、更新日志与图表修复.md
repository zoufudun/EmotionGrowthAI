# EmotionGrowth AI 学生端数据隔离、新手引导、测试账户清空、更新日志与图表修复

## 背景

用户提出 5 项改进需求：

1. **数据隔离**：不同学生账号登录后能看到其他人的打卡记录、反思日记等，需要按用户隔离。
2. **新注册账号操作引导**：为新用户增加一个步进式引导，帮助他们了解系统功能。
3. **默认账户清空数据**：默认 `student` 测试账号登录后，自动清空该用户的所有数据，方便测试。
4. **更新日志与用户反馈**：在"关于系统"页面增加更新日志和用户反馈入口。
5. **ECharts 曲线图 resize bug**：从"成长日记与今日打卡"Tab 切回"个人成长面板"时，ECharts 图表没有正确 resize，曲线缩在一起。

---

## 问题分析

### 需求 1：数据隔离

当前 localStorage 中以下 key **没有** 按用户区分，导致不同学生账号登录后看到同一份数据：

| localStorage Key | 使用位置 | 问题 |
|---|---|---|
| `moodCheckInHistory` | StudentDashboard（打卡历史） | ❌ 全局共享 |
| `reflectiveLogs` | StudentCounseling（反思日记） | ❌ 全局共享 |
| `auditLogs` | App.jsx（操作审计日志） | ❌ 全局共享（学生端显示） |

已有用户级隔离的 key（不需要修改）：
- `diaryDraft_${username}` ✅
- `testDraft_${username}` ✅
- `musicFavorites_${username}` ✅
- `musicRelaxationLogs_${username}` ✅
- `musicReminderActive_${username}` ✅
- `assignedTasks` 通过 `studentName` 字段过滤 ✅
- `assessmentRecords` 通过 `studentName` 字段过滤 ✅

**修复方案**：将上述 3 个全局 key 改为用户级 key（`moodCheckInHistory_${userId}` 等），或者在加载时按 `userId / username` 过滤。考虑到 `moodCheckInHistory` 和 `reflectiveLogs` 是纯个人数据，使用 **带 userId 后缀的 localStorage key** 最为清晰。审计日志 `auditLogs` 学生端应只显示自己的日志。

### 需求 2：新手引导

使用 Ant Design 的 `Tour` 组件（antd v5.1+ 内置），在新注册用户首次登录时自动弹出分步引导，引导他们了解侧边菜单的各个功能入口。通过 `localStorage` 标记 `onboardingDone_${userId}` 来避免重复显示。

### 需求 3：默认测试账户清空数据

当检测到登录的是默认测试账户（username === 'student'），在登录完成后自动清除该用户在 localStorage 中的所有关联数据（打卡记录、反思日记、测试草稿、音乐收藏等）。

### 需求 4：更新日志 & 用户反馈

在 [About.jsx](file:///c:/Users/Phodon/psychology-admin/src/views/About.jsx) 页面中增加两个新卡片：
- **更新日志**：展示系统版本更新记录（硬编码的版本日志列表）
- **用户反馈**：提供反馈输入表单，保存到 localStorage 并展示已提交反馈列表

### 需求 5：ECharts resize bug

**根本原因**：在 [StudentDashboard.jsx](file:///c:/Users/Phodon/psychology-admin/src/views/StudentDashboard.jsx#L602-L706) 中，ECharts 的 `useEffect` 依赖 `[activeTabKey, checkInHistory]`。当用户从 `daily-log` Tab 切回 `trends` Tab 时，ECharts 图表 DOM 元素由隐藏变为显示，但 chart 被重新 `init`，此时 DOM 容器宽度可能还是 0（Tab 切换动画未完成）。需要在 chart init 后额外调用 `setTimeout(() => myChart.resize(), 0)` 来确保在 DOM 布局完成后进行一次 resize。此外，使用 `ResizeObserver` 替代 `window.resize` 事件可以更精准地监听容器变化。

2. 使用 `ResizeObserver` 替代 `window.addEventListener('resize')` 来精准监听容器尺寸变化：
   ```js
   const resizeObserver = new ResizeObserver(() => {

   })
   resizeObserver.observe(chartDom)
   ```
